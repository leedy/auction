# Beaker's Feature Requests for Auction Monitor

These requests are designed to improve the synergy between Beaker (OpenClaw Assistant) and the Auction Monitor application/MCP server.

## 1. Vision-Enhanced Evaluation (Priority: High)
Beaker has vision capabilities. To use them, the MCP server needs to provide the imagery.
- **Requirement:** Update `get_lot_details` and `list_lots` MCP tools to include the primary `image` URL.
- **Goal:** Allow Beaker to visually inspect lot photos (e.g., checking the bottom of a cast iron pan for a Griswold logo or inspecting the corners of a comic book) before confirming a flag.

## 2. Feedback Loop Integration (Priority: High)
The "Not Interested" button in the UI should be more than a visual filter; it should be a training signal.
- **Requirement:** Update the evaluation script to check for `userFeedback: "not_interested"` in existing records.
- **Logic:** If an item with specific keywords (e.g., "Food Processor", "Modern Appliance") is marked as not interested, the AI evaluation pass should weight those keywords negatively in future runs.
- **Safety:** Ensure manual flags (`interested: true`) are never overwritten by automated keyword passes, as discussed previously.

## 3. Recovery Logistics: "Pickup Queue" (Priority: Medium)
David is currently recovering from surgery and has limited mobility.
- **Requirement:** Add a `needsPickup` boolean field to the Evaluation schema and a corresponding toggle in the UI.
- **Usage:** When David is interested in an item but realizes it's heavy/bulky (like a #10 Dutch Oven), he can toggle "Needs Pickup."
- **Export:** Create a tool or endpoint to generate a "Pickup List" that David can share with his support network.

## 4. MCP Tool for Evaluation Reasoning
To avoid Beaker "hallucinating" or over-guessing, the MCP server should allow Beaker to provide structured reasoning.
- **Requirement:** The `save_evaluation` tool should accept a `reasoning` string (which it currently does, but let's ensure it's prominently displayed in the UI).
- **Vibe:** David values transparency. Beaker should always explain *why* a specialist ("The Picker") thinks an item is special.

## 5. "Special Interest" Category Mapping
- **Requirement:** Allow the interest profile (currently in `.yaml`) to be mapped to specific UI categories.
- **Goal:** Rather than just "Interested," show items under "Marvel Collection," "Cast Iron Kitchen," or "70s Nostalgia."
